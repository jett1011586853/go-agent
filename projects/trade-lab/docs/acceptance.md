# Trade-Lab 验收文档

## 项目概述

Trade-Lab 是一个事件驱动量化回测与实时风控平台，采用 Go 单体架构，支持 SQLite/PostgreSQL 数据库切换。

## 验收标准

### 功能验收

| 功能模块 | 状态 | 说明 |
|---------|------|------|
| 配置管理 | ✅ 通过 | 支持 YAML + 环境变量 + profile 切换 |
| 日志系统 | ✅ 通过 | zap 结构化日志，console/json 格式 |
| 错误码体系 | ✅ 通过 | 1xxx-6xxx 分段定义 |
| 健康检查 | ✅ 通过 | /health, /ready 端点 |
| 事件总线 | ✅ 通过 | InMemoryBus，支持 Publish/Subscribe/Replay |
| 行情回放 | ✅ 通过 | CSV 格式 K线/Tick 数据回放 |
| 策略引擎 | ✅ 通过 | SMA/布林带/动量三种策略 |
| 回测引擎 | ✅ 通过 | 支持手续费、滑点、仓位管理 |
| 参数优化 | ✅ 通过 | GridSearch 网格搜索 |
| 订单撮合 | ✅ 通过 | 市价/限价订单模拟撮合 |
| 风控系统 | ✅ 通过 | 单笔限额、日损、持仓上限、熔断 |
| 审计日志 | ✅ 通过 | 所有订单审批/拒绝决策记录 |
| REST API | ✅ 通过 | 策略/任务/风控/仪表盘端点 |
| 故障注入 | ✅ 通过 | DB锁/消息积压/坏数据/超时 |
| 容器化 | ✅ 通过 | Dockerfile + docker-compose |
